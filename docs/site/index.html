What makes a PR review faster

Static-analysis tools tell you which line has a problem. ArchiteX tells you what changed in your architecture and why a reviewer should care.

45 AWS resource types

VPC, EC2, S3, IAM, Lambda, API Gateway, CloudFront, KMS, SNS/SQS, NACL, EBS, ECS, EKS, Auto Scaling, RDS groups. Tracking the architectural primitives that matter, not every CRUD-style resource.

18 deterministic risk rules

Each rule has a fixed weight, a stable ID, and a one-line explanation. No probabilistic scoring, no model calls, no hidden config. Same diff → same score, every time.

0 SaaS, telemetry, accounts

Analysis runs in your GitHub Actions runner. The only network call is the PR comment to the GitHub REST API using the workflow’s own GITHUB_TOKEN.

Library mode for module authors

Resources gated behind count = var.create ? 1 : 0 render as conditional phantoms in the diagram. Risk rules refuse to score them — no false positives on conditional gates.

Self-contained HTML report

Every run produces a report.html with no JavaScript, no CDN, no remote fonts. Open it in an air-gapped browser and see the full review with severity badges, reasons, suppressions, the Mermaid diagram, and a SHA-256 manifest.

Baseline anomaly detection

Optional .architex/baseline.json records the kinds of resources your repo has ever contained — no raw HCL, no names. New resource types show up as low-weight signals that layer on top of the existing rules.

How it works

A strict four-stage pipeline. Every stage is deterministic. The Interpreter is template-based — no inference, no model calls, no network.

base/*.tf  ─┐
            ├─> 1. Fact-Finder  ──>  2. Delta Engine  ──>  3. Risk & Policy  ──>  4. Interpreter  ──>  Sticky PR comment
head/*.tf  ─┘    HCL parse           added · removed         18 rules                Mermaid · MD            +
                 graph build         · changed               weighted score          HTML · audit            Audit bundle
                                                                                                             (manifest.json
                                                                                                             with SHA-256)

30-second quickstart

Drop this into your Terraform repo at .github/workflows/architex.yml and you’re done.

name: ArchiteX
on: pull_request
permissions:
  contents: read
  pull-requests: write
jobs:
  architex:
    runs-on: ubuntu-latest
    steps:
      - uses: danilotrix86/ArchiteX@v1.3.0
        with:
          terraform-dir: infra

Open a PR that touches infra/*.tf. ArchiteX posts a sticky comment with the score, the diagram, and the reasons. Add mode: blocking to fail the build on high-risk changes.
